## Key Takeaways

- A single skill file hits a ceiling quickly; real domain depth requires a network of interconnected skill files (a "skill graph") linked with wikilinks that carry semantic meaning in prose.
- Progressive disclosure (index -> descriptions -> links -> sections -> full content) lets agents make most routing decisions before reading any full file, keeping context usage efficient.
- The three core primitives are wikilinks embedded in prose, YAML frontmatter descriptions for scanning, and Maps of Content (MOCs) that organize clusters of related skills into navigable sub-topics.
- Skill graphs apply the same discovery pattern recursively: each node is a standalone claim or technique, and links between nodes let the agent traverse as deep as the domain requires.
- Practical applications span trading, legal, company knowledge, therapy, and any domain too rich for a single file — the graph structure makes breadth and depth coexist.
- Building a skill graph starts with an index file that acts as an entry point directing agent attention, not a flat lookup table.
- The shift from single skills to skill graphs is the difference between an agent that follows instructions and one that understands a domain.

## One-Line Summary

Connecting many small, standalone skill files into a wikilink-based graph with progressive disclosure lets AI agents navigate deep domain knowledge instead of being limited to what fits in a single context injection.
