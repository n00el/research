## Key Takeaways

- **Agents are the new primary software users.** Enterprises may have 100X-1,000X more agents than employees, making agents the dominant consumer of all software. Products must be redesigned around this reality.

- **"Make something agents want" is the new PG mantra.** Just as Paul Graham advised "make something people want," software companies must now optimize for agent adoption -- agents won't attend webinars or see ads, they'll just pick the best tool for the job.

- **API-first is non-negotiable.** If a feature doesn't have an API, it might as well not exist. CLI exposure, MCP servers, and clean API design are now as critical as UX design was for human users. Even account signup must be API-accessible.

- **Business models must evolve to consumption-based.** Seat-based pricing breaks down when agents do hours of human-equivalent work in seconds. Software needs volume/consumption-based models, including letting agents make their own payments.

- **A new infrastructure stack is emerging for agents.** Sandboxed compute (E2B, Daytona, Modal, Cloudflare), agent-native file systems, agent identities and mailboxes (Agentmail), agent-optimized web search (Exa), and agent wallets (Stripe, Coinbase) form an entirely new tech layer.

- **Security and governance become critical at agent scale.** Agents working on sensitive, regulated workflows need their own identities, authentication, access controls, and audit trails -- an entire governance stack parallel to what exists for humans today.

- **Agents expand the frontier of what gets done.** Beyond automating existing tasks, agents enable running previously unaffordable simulations, prototyping every idea with multiple options, and reviewing all data instead of sampling.

## One-Line Summary

Aaron Levie argues that software must be fundamentally redesigned -- API-first, with new business models, infrastructure, identity, and governance -- because trillions of AI agents will soon become the primary users of all enterprise software.
