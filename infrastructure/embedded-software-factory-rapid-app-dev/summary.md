# A sneak preview behind an embedded software factory. I suspect rapid application dev is back.

**Author:** geoff (@GeoffreyHuntley)
**Date:** 2026-03-08
**Source:** https://x.com/GeoffreyHuntley/status/2030683143360119292

## Key Takeaways

- Geoffrey Huntley is building Latent Patterns with an "embedded software factory" where the product itself is the IDE -- he uses a "designer mode" within the app to develop the product from within the product, then ships changes via Cursor Cloud Agents with a risk-based deployment approach.
- He argues we are returning to the Microsoft Access / Delphi / Visual Basic era of hyper-personalised business software, where every company had custom-built tools instead of stitching together SaaS with workflow automations like Zapier.
- Every business needs the same core widgets (Analytics, CRM, Support Desk, Newsletters, Meeting Scheduling) and these should be first-party, not separate third-party tools glued together.
- He uses @peopledatalabs for automatic customer enrichment (job title, salary estimates, decision-making power) and runs Perplexity searches to build deep customer profiles, all stored first-party in his data tables.
- His meeting workflow includes a custom transcription bot that joins calls, then runs Challenger-based sales and SPIN Selling analysis via LLM prompts to extract competitive landscape, buying signals, pain points, and follow-up items.
- His philosophy on AI-assisted development: "I'm on the loop, not in the loop" -- he watches agents build from his phone rather than manually coding, but emphasizes watching the inferencing loops is where the learning happens.
- CI/CD as we know it is too slow; the goal is live-editing the program's memory directly, shipping instantly with risk-gating only for high-risk changes like database migrations.

## One-Line Summary

Geoffrey Huntley demonstrates how Latent Patterns functions as its own IDE with AI-powered live editing, risk-based auto-deployment via Cursor Cloud Agents, and first-party CRM/analytics/scheduling -- arguing this is the return of rapid application development that businesses had in the Microsoft Access era.
