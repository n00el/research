# 8 Principles of Harness Engineering for AI Coding Agents

**Author:** Muratcan Koylan (@koylanai)
**Date:** March 5, 2026
**Source:** https://x.com/koylanai/status/2029427980699644360

## Key Takeaways

- Engineers become environment designers — the role shifts from writing code to designing agent scaffolding
- Progressive disclosure beats monolithic documentation; use a concise index linking to layered reference materials
- Mechanical enforcement (linters, CI) is more reliable than natural-language instructions for agents
- Velocity-first merge philosophy: address test failures post-merge rather than blocking on them
- Agent-to-agent code review with humans only for edge cases
- All relevant information must be repo-encoded and version-controlled for agent accessibility

## One-Line Summary

Eight principles from OpenAI's harness engineering approach that redefine the engineer's role from code writer to environment designer for AI coding agents.
