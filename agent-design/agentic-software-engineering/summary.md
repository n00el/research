# Agentic Software Engineering

**Author:** Ashpreet Bedi (@ashpreetbedi)
**Date:** March 6, 2026
**Source:** https://x.com/ashpreetbedi/status/2029953139856531528

## Key Takeaways

- Building agents is easy; running them in production is a distributed systems problem
- Three-part framework: Build (model/tools/memory) → Serve (scalable API) → Connect (where users are)
- Six pillars: Durability, Isolation, Governance, Persistence, Scale, Composability
- Agent engineering = 40% agent dev + 40% system design + 20% security engineering
- Most products stall at the "Serve" stage — horizontal scaling, streaming, retry logic
- Agents should be structured as standard API services for composability

## One-Line Summary

Production agent engineering requires treating agents as distributed systems with six pillars (durability, isolation, governance, persistence, scale, composability), not as simple scripts.
