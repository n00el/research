# How To Be A World-Class Agentic Engineer

**Author:** sysls (@systematicls)
**Date:** March 3, 2026
**Source:** https://x.com/systematicls/status/2028814227004395561

## Key Takeaways

- Keep it simple: barebones CLI + core principles outperforms over-tooled setups; frontier companies absorb useful external solutions
- Context is everything: give agents only the exact information they need, nothing more
- Separate research from implementation; use fresh agents with clean context for each
- Exploit sycophancy via adversarial multi-agent pattern (bug-finder + disprover + referee)
- Use task contracts with deterministic tests as completion milestones — agents struggle knowing when they're done
- CLAUDE.md should be a minimal conditional directory ("if X, read this"), not a monolith
- One fresh session per contract beats long-running sessions that accumulate context bloat

## One-Line Summary

World-class agentic engineering is about minimalist tooling, precise context management, and disciplined task contracts — not complex harnesses and plugins.
