# Summary: I want to learn how to build AI Agents with Claude (full course)

## Key Takeaways

- **Agent vs. Chat distinction:** Chat is single-turn Q&A; an agent autonomously plans, executes, observes results, and iterates until a task is complete (Think -> Act -> Observe -> Repeat loop).
- **The Claude Agent Stack has four layers:** Claude Code (terminal agent), Claude Agent SDK (Python/TS library), MCP (Model Context Protocol for external tool connections), and Agent Teams (multi-agent parallel orchestration).
- **Claude Code is the fastest entry point:** No SDK or custom code needed -- it can read files, write code, run bash, commit to Git, review PRs, and manage entire projects from the terminal.
- **MCP connects agents to everything:** Over 200 community MCP servers exist as of early 2026, enabling integrations with GitHub, databases, browsers, Slack, and more.
- **Agent Teams enable parallelism:** Multiple specialized agents (API, frontend, reviewer) work simultaneously and coordinate through shared task lists.
- **Decision framework for choosing agent type:** Single question = chat; multi-step with tools = agent; parallel specialist work = multi-agent.
- **Article is partially paywalled:** Only Module 1 (what agents are) and the start of Module 2 (Claude Code setup) are publicly available; Modules 3-5 (Agent SDK, multi-agent orchestration, production deployment) require X Premium.
