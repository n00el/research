# How to Use Claude Cowork Like a Second Employee

**Author:** Jesse Pujji (@jspujji)
**Date:** March 19, 2026
**Source:** https://x.com/jspujji/status/2034616093386678778

## Key Takeaways

- Cowork is fundamentally different from Chat: you delegate tasks and come back to finished files, rather than driving a back-and-forth conversation
- Set up a dedicated workspace with Context, Projects, and Outputs subfolders to keep Claude sandboxed and your files safe
- Build three context files (who-i-am.md, how-i-talk.md, how-you-work.md) so Cowork knows your voice and working style from the first session
- Use Global Instructions to define guardrails and preferences that persist across all Cowork sessions
- The three essential plugins for marketers are Productivity, Marketing, and Design -- each bundles skills, connectors, and slash commands into ready-to-use workflows
- Practical use cases include file organization, micro-influencer sourcing, and scheduled weekly competitor analysis

## One-Line Summary

A practical setup guide for non-technical marketers to configure Claude Cowork as a delegate-and-forget system using context files, global instructions, plugins, and real workflow examples.
