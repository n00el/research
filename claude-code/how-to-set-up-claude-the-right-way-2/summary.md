# How to set up Claude the right way (so you actually stop going back to ChatGPT)

**Author:** Ruben Hassid (@rubenhassid)
**Date:** February 20, 2026
**Source:** https://x.com/rubenhassid/status/2024871775574593798

## Key Takeaways

- Claude in 2026 is not just a chatbot -- it is six distinct tools: Cowork (desktop app), Model (Opus 4.6 + Extended Thinking), Excel add-in, Plugins (specialist skill packs), Artifacts (interactive outputs), and Projects (persistent context folders).
- **Cowork is the most important feature:** it reads your local files before responding, turning context-rich .md files into a replacement for complex prompting. The more context files you provide, the less prompting you need.
- The recommended first prompt pattern is to ask Claude to read files first, then ask clarifying questions before executing -- forcing it to understand intent before generating output.
- **Opus 4.6 with Extended Thinking enabled** is the recommended model configuration (released February 5, 2026). Extended Thinking forces Claude to reason before responding.
- Claude Connectors integrate with 50+ tools (Slack, Google Drive, Notion, Figma) allowing it to reference your existing work context mid-conversation.
- Claude has clear limitations: no image generation (use Gemini), no video generation (use Seedance 2.0), and weaker real-time search (use Grok). The author advocates using the right tool for each task rather than forcing one tool for everything.
- The mindset shift from ChatGPT to Claude is moving from "write better prompts" to "write better context files" -- .md files that describe your writing style, brand rules, and best examples compound in value over time.

## One-Line Summary

Claude's power in 2026 comes not from better prompting but from feeding it rich context files through Cowork, making Opus 4.6 + Extended Thinking the go-to setup for knowledge workers switching from ChatGPT.
