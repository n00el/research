# The Claude Dispatch Guide: 48 Hours Running AI From My Phone

**Author:** Pawel Huryn (@PawelHuryn)
**Date:** Mar 23, 2026
**Source:** https://x.com/PawelHuryn/status/2036058594433519790

## Key Takeaways

- Dispatch is an orchestrator, not a chat interface -- from your phone you spawn and manage multiple parallel Cowork task sessions running on your desktop, each with its own context and connectors.
- The real productivity shift is not "filling dead time" but restructuring your day around async direction: 25 minutes of phone-based direction yielded 3+ hours of parallel AI execution across 60+ task sessions over 48 hours.
- Sync your Cowork workspace folder with Google Drive to solve both the file input and output problem -- everything Claude saves appears on your phone, and vice versa.
- Always have Dispatch read your CLAUDE.md before delegating tasks; otherwise the orchestrator formulates imprecise instructions for subtasks.
- Anthropic shipped 4 remote access surfaces in 5 months (Dispatch, Channels, Remote Control, Web Sessions) -- store your knowledge layer in a GitHub repo so it compounds across all surfaces rather than being locked to one.
- The highest-leverage investment for PMs isn't learning new AI tools -- it's building the judgment, knowledge systems (CLAUDE.md, skill libraries, workflow patterns), and domain expertise that make you a better orchestrator of whatever tool comes next.
- Each subtask requires separate folder access approval on desktop -- there's no workaround, it's the security model. Plan accordingly before going mobile.

## One-Line Summary

Claude Dispatch turns your phone into an async command chair for parallel AI work sessions on your desktop, but its real value only compounds when backed by a structured knowledge system (CLAUDE.md + skill libraries) that persists across all Claude surfaces.
