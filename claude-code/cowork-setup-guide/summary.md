# How to Set Up Claude Cowork the Right Way (So It Actually Does Your Work While You Step Away)

**Author:** Nav Toor (@heynavtoor)
**Date:** Feb 25, 2026
**Source:** https://x.com/heynavtoor/status/2026717574776631556

## Key Takeaways

- Cowork's biggest leverage comes from preparing context files (about-me.md, brand-voice.md, working-style.md) rather than crafting better prompts — context compounds while prompt skill depreciates.
- Appending "DO NOT start working yet. First, ask me clarifying questions" to prompts forces Cowork to use its AskUserQuestion feature, dramatically improving output quality on the first pass.
- The plugin ecosystem already has enough market impact to move public equities — Thomson Reuters dropped 16% and LegalZoom dropped 20% when the Legal plugin launched.
- Cowork has no cross-session memory, so durable context must be maintained in markdown files and global instructions that auto-load each session.
- Folder-level instructions enable per-client or per-project configuration that loads automatically, making it practical for consulting and agency workflows.
- Tasks die if you close the desktop app but survive sleep mode — a non-obvious operational detail that matters for long-running autonomous work.
- The author reports covering roughly 60% of knowledge work with a simple loop: describe the task, answer three clarifying questions, and return 20 minutes later to a finished deliverable.

## One-Line Summary

Cowork's real power is not in prompting but in setting up persistent context files, folder instructions, and plugins that let Claude operate autonomously on your desktop while you step away.
