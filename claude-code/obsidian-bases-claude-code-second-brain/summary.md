# How I Organize My Second Brain With Obsidian Bases and Claude Code

**Author:** Noah Vincent (@noahvnct)
**Date:** Mar 20, 2026
**Source:** https://x.com/noahvnct/status/2035013355530772791

## Key Takeaways

- Folder-based systems like PARA impose a hidden "filing tax" on every note -- forcing you to decide where it belongs before storing it, and then remember that decision on retrieval
- Obsidian Bases enables a flat-file architecture (all notes in one folder) with dynamic Category and Subject container notes that surface content from any angle via YAML frontmatter wikilinks
- The 5-folder structure (Inbox, Notes, Categories, Subjects, System) replaces hierarchical organization with a "what is this note?" model instead of "where does this go?"
- Claude Code acts as an automated maintenance layer via CLAUDE.md files in every folder -- it processes inbox items, assigns categories/subjects, creates properly templated notes, and bulk-updates metadata
- The combination of Obsidian Bases for visual organization + Claude Code for automated maintenance means the knowledge system never degrades from neglect
- Content creation is supercharged because Claude has full context over the entire vault, synthesizing relevant notes rather than starting from generic prompts

## One-Line Summary

A flat-file Obsidian vault using Bases for multi-dimensional navigation (Categories + Subjects) combined with Claude Code for automated maintenance eliminates the filing friction that kills every folder-based second brain system.
