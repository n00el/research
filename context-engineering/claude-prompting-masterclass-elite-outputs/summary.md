## Key Takeaways

- Three biggest prompting mistakes: no context (generic input = generic output), being too vague (Claude is a reasoning model not a search engine), and not iterating (great outputs rarely come from a single prompt)
- Five essential prompt components from Anthropic: task context (role + task), background data, detailed rules, examples (using `<example>` tags -- the single biggest quality unlock), and output formatting
- XML/JSON structured prompting is the biggest upgrade for LLM inputs -- it speaks Claude's native language since it was trained on XML tags
- Reverse prompting (asking Claude to question you first) is the most underrated technique, especially when approaching new task types
- Chain prompting breaks big tasks into connected smaller prompts where each step carries full context from the previous one, compounding quality
- Anthropic recommends: put longform data at the top of prompts, use scratchpads for reasoning, have Claude self-check before finishing, and define explicit success criteria
